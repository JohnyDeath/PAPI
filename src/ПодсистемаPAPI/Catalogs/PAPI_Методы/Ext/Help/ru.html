Справочник Методы (PAPI_Методы)

Позволяет описать метод, параметры вызова, алгоритм решения и формат ответа. 

Реквизиты:

Разрешен – Булево – Разрешает выполнение метода, в отличии от алгоритма проверяется до функции выполнения, данная особенность позволяет тестировать запрещенные методы. Если метод запрещен значение реквизита ложь. При запрете выдаст ошибку с кодом "Err4", См. Справочник.PAPI_Сообщения.

ВерсияМетода – Строка (9) – используется в шаблоне обращения к методу, по умолчанию V1. Шаблон = /{ВерсияМетода}/{ИмяМетода}

ИмяМетода – Строка (150) – Ключ поиска метода. Используется в шаблоне. Шаблон = /{ВерсияМетода}/{ИмяМетода}

Комментарий – Строка – Информация по методу.

ЛогироватьЗапрос – Булево – Указывает на необходимость записи данных запроса в документ PAPI_ВходящийЗапрос.

ЛогироватьМетод – Булево – Указывает на необходимость записи лога в регистр сведений PAPI_ЛогМетодов.

Привилегированный – Булево – Признак запуска метода в привилегированном режиме.

Настройки – ХранилищеЗначенией – Содержит структуру метода, состоящую из кода произвольного алгоритма, кода для обработки тела запроса и параметров запроса.

ПроизвольныйАлгоритм – Булево – Указывает на то, как метод должен выполняться. Выполняется либо алгоритм, либо произвольный код.

Алгоритм – Справочник.PAPI_Алгоритмы- Если ПроизвольныйАлгоритм = Ложь, тогда метод работает по указанному алгоритму.

Стандартный – Булево - Метод создан PAPI.

ТелоЗапроса - Перечисление.PAPI_ТелоЗапроса – Тип данных в теле запроса (Строка, ДвоичныеДанные, Поток).

ТелоОтвета - Перечисление.PAPI_ТелоОтвета – Тип данных в теле ответа.

Табличные части:

            HTTPМетоды – содержит разрешенные методы (GET, POST, HEAD, PUT, DELETE, PATCH, MERGE, OPTIONS, TRACE, CONNECT).

            ЗаголовкиОтвета – содержит заголовки ответа.

Пример URL:

http[s]://host/{base}/hs/papi/{MethodVersion}/{MethodName}
